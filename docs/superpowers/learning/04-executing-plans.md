# 第四章：执行计划 (executing-plans)

## 何时使用
- 已有书面的实现计划
- 需要在单独会话中执行计划
- 需要审查检查点

## 两种执行模式

### 1. 子代理驱动（推荐）
- 每个任务由新鲜的子代理执行
- 任务之间有审查检查点
- 快速迭代
- 适用：大多数实现任务

### 2. 内联执行
- 在当前会话中使用 executing-plans
- 批量执行带检查点
- 适用：需要人工介入的任务

## 使用流程

### 子代理驱动模式
```
1. 确保有完整的书面计划
2. 调用 Skill: superpowers:subagent-driven-development
3. 每个任务自动分派给子代理
4. 自动进行两轮审查（规格 + 质量）
5. 任务完成后继续下一个
```

### 内联执行模式
```
1. 确保有完整的书面计划
2. 调用 Skill: superpowers:executing-plans
3. 按任务顺序执行
4. 每个任务后进行审查
```

## 示例

### 计划完成后选择执行模式
```
计划已完成并保存到 `docs/superpowers/plans/xxx.md`。两种执行选项：

1. **子代理驱动（推荐）** - 每个任务由新鲜子代理执行，任务间自动审查
2. **内联执行** - 在当前会话中批量执行，需要人工确认

选择哪种方式？
```

### 执行过程示例
```
Task 1: 基础实现
  [子代理执行中...]
  ✅ 规格审查通过
  ✅ 代码质量审查通过
  ✅ 提交：abc123

Task 2: 添加验证
  [子代理执行中...]
  ...
```
