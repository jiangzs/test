# 第八章：子代理协作 (dispatching-parallel-agents)

## 何时使用
- 面对 2+ 个独立任务
- 任务可以并行执行
- 任务之间没有共享状态或顺序依赖

## 核心优势
- 并行执行加快进度
- 独立上下文避免污染
- 专门代理专注任务

## 使用流程

```
1. 识别独立任务
2. 调用 `Skill: superpowers:dispatching-parallel-agents`
3. 每个子代理处理一个任务
4. 合并结果
```

## 示例场景

### 场景 1: 同时运行测试和构建
```
独立任务：
- 任务 A: 运行单元测试
- 任务 B: 运行 lint 检查
- 任务 C: 构建文档

使用 dispatching-parallel-agents 并行执行
```

### 场景 2: 多文件修改
```
独立任务：
- 任务 A: 更新用户模块
- 任务 B: 更新订单模块
- 任务 C: 更新产品模块

三个模块互不依赖，可并行修改
```

## 任务依赖判断

### 可以并行的任务
- 修改不同文件
- 无共享状态
- 无数据依赖

### 需要串行的任务
- 任务 B 依赖任务 A 的输出
- 修改同一文件的不同部分
- 共享全局状态变更

## 示例代码

### 并行执行
```python
# 错误：串行执行
result_a = task_a()  # 等待完成
result_b = task_b()  # 等待完成
result_c = task_c()  # 等待完成

# 正确：并行执行
with parallel():
    dispatch(task_a)
    dispatch(task_b)
    dispatch(task_c)
```

## 最佳实践

### 任务分解
- 确保任务真正独立
- 明确每个任务的输入输出
- 预设合并策略

### 结果合并
- 检查冲突
- 验证完整性
- 统一提交
