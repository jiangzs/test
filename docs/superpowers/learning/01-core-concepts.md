# 第一章：核心概念与规则

## Superpowers 是什么？

Superpowers 是一套为 Claude Code 设计的技能系统，帮助你：
- 在正确的时间做正确的事
- 遵循经过验证的开发流程
- 避免常见的开发陷阱
- 提高代码质量和开发效率

## 核心规则

### 规则 1: 1% 规则
> 如果认为某个技能有 1% 的可能性适用，你绝对必须调用它。

这不是可选的。这是强制性的。

### 规则 2：技能优先级
1. 流程技能优先（头脑风暴、调试）- 决定如何 Approach 任务
2. 实现技能其次（前端设计、MCP 构建器）- 指导执行

### 规则 3：用户指令最高优先级
用户指令 > 技能规则 > 默认系统行为

## 技能调用流程

```
用户消息 → 是否有技能适用？→ 调用 Skill 工具 → 宣布用途 → 遵循技能执行
                              ↓
                          创建 TodoWrite（如有清单）
```

## 常见误区

| 错误想法 | 正确做法 |
|----------|----------|
| "这只是个简单问题" | 任何行动都是任务，先检查技能 |
| "我需要先收集上下文" | 技能告诉你 HOW 收集，先调用技能 |
| "技能会拖慢进度" | 无纪律的行动浪费时间，技能防止这个 |
| "我记得这个技能" | 技能会演进，阅读当前版本 |
