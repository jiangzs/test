# 第二章：头脑风暴 (brainstorming)

## 何时使用
- 开始任何创意工作之前
- 创建新功能、构建组件、添加功能或修改行为
- 需要探索用户意图、需求和设计

## 触发条件
- 用户说"让我们构建 X"
- 用户说"我想添加一个功能..."
- 任何需要设计决策的任务

## 使用流程

1. 宣布：`Using brainstorming to explore [topic]`
2. 调用 `Skill: superpowers:brainstorming`
3. 跟随技能的引导进行探索

## 示例场景

### 场景：构建待办事项应用
```
用户："我想创建一个待办事项应用"

你：我将使用 brainstorming 技能来探索需求。
[调用 Skill: superpowers:brainstorming]
```

### 场景：添加用户认证
```
用户："我们需要添加用户登录功能"

你：让我使用 brainstorming 来探索认证方案。
[调用 Skill: superpowers:brainstorming]
```

## 与其他技能的关系
- brainstorming → writing-plans → executing-plans
- 头脑风暴完成后，通常会产生一个实现计划

## 练习
1. 选择一个你想构建的功能
2. 使用 brainstorming 技能探索需求
3. 记录探索结果
